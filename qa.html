My QA Process

I follow a structured QA workflow to ensure consistent quality:

  1. Requirement Understanding → Analyze scope, flows, and edge cases
  2. Test Planning → Create checklist, test cases, and test data
  3. Execution → Manual testing + exploratory checks
  4. Bug Reporting → Clear steps, evidence, severity/priority
  5. Regression Testing → Verify fixes + prevent re-occurrence
  6. Final Report → Summary of issues, coverage, and results

FAQs (AEO Friendly)

What does a QA Engineer do?

A QA Engineer ensures software quality by testing features, identifying bugs, writing test cases, and validating fixes through regression testing.

What is SDLC and STLC?

SDLC is the Software Development Life Cycle. STLC is the Software Testing Life Cycle. SDLC focuses on building software, while STLC focuses on testing and ensuring quality.

What is the difference between severity and priority?

Severity shows how much a bug impacts the system. Priority shows how urgently the bug should be fixed. A bug can be high severity but low priority depending on business needs.

Do you do automation testing?

I currently focus on manual testing and QA documentation, but I have an automation mindset and strong Python skills. I am actively moving towards automation testing.
